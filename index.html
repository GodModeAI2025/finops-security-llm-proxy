LLM API Proxy

Ein selbstlernendes API-Gateway, das echte LLM-API-Keys kapselt, eigene Proxy-Tokens vergibt und Budgets automatisch via P90-Analyse optimiert — deployed auf drei Cloud-Plattformen.

Kernfunktionen

Enterprise-Features in einem experimentellen Proof-of-Concept

P90

Selbstlernende Budgets

Limits werden automatisch aus historischen Sessions berechnet. Ab 5 Datenpunkten greift die P90-Perzentilanalyse mit 10% Sicherheitsmarge.

⚠

Auto-Revocation

Tokens werden automatisch revoziert bei Budget-Limit, TTL-Ablauf oder Fehler-Streak — serverseitig bei jedem Request geprüft.

$

Exakte Kostenanalyse

Pro Request: Input/Output-Tokens × Provider-Preis. Aggregiert als Min/Max/Durchschnitt/Median/P90 pro Themenfeld.

⇄

Multi-Provider Proxy

Anthropic (Claude), OpenAI (GPT-4o), Google (Gemini) — einheitliche API, transparenter Provider-Wechsel per Model-Prefix.

▶

Streaming-Support

SSE-Streaming mit Usage-Extraktion aus dem Stream (GCP & Cloudflare). Echtzeit-Antworten mit korrekter Kostenerfassung.

☁

Adaptive Budgetierung

Das System lernt aus jeder Session, welche Ressourcen ein Themenfeld tatsächlich benötigt

Stufe 1

Unbekanntes Thema

Keine historischen Daten. Defaults: max 50 min, $50 Budget.

Stufe 2

1–4 Sessions

Limit = Max(bisherige Werte) × 1,10 Sicherheitsmarge.

Stufe 3

5+ Sessions

Limit = P90-Perzentil × 1,10. Outlier werden ignoriert.

Electron PoC Client

Demo-Client zum Testen aller Proxy-Funktionen

Vollständiger Test-Client

Der Electron-Client bietet eine 5-Screen-Oberfläche zum interaktiven Testen aller Backend-Varianten — inklusive Live-Monitoring von Budget und TTL.

  • Backend-Preset-Wechsel (GCP / Cloudflare / AWS)
  • Live-Timer und Budget-Countdown
  • Adaptive Limits aus Server-Antwort
  • Session-Feedback (Erfolg/Fehler)
  • Statistik-Dashboard mit P90-Analyse
  • Request-Logging und Fehlerdiagnose
# Quickstart $ cd client/electron-app $ npm install $ npm start # Die App startet mit: 1. Proxy-URL eingeben 2. Admin-Key setzen 3. Themenfeld wählen (z.B. "code-review") 4. Chat starten — Timer + Budget live 5. Session beenden → Messpunkt gespeichert 6. Nächste Session → Limits adaptiv

Quickstart

Ein Backend aussuchen, deployen, Client starten

01 GCP Cloud Run

cd backends/gcp-cloud-run
npm install
chmod +x deploy.sh
./deploy.sh

02 Cloudflare Workers

cd backends/cloudflare-workers
npm install
npx wrangler deploy

03 AWS Lambda

cd backends/aws-lambda
npm install
npx cdk deploy
